-git init -> კონკრეტულ საქაღალდეში ქმნის git ის რეპოზიტორიას 

ხდება ასეთი რაღაც :

-ქმნის დამალულ საქაღალდეს (.git)

-ამ საქაღალდეში ინახება რაც საჭიროა Git ისთვის:ცვლილებები, მისი ისტორია ,branch_ები და ა.შ

-რეპოზიტორიის შექმნის მომენტიდან Git იწყებს ამ პროექტში ცვლილებების დაკვირვებას


---------------------------------------------------------------------



-git status -> არის ბრძანება რომელიც გვიჩვენებს Git ის მიმდინარე მდგომარეობას - ანუ რა ხდება ახლა პროექტში:რომელი ფაილები შეიცვალა,რომლებია მზად commit-ისთვის და რომლები ჯერ არა


1)git status აჩვენებს თუ რომელ branch ზე ხარ:

მაგ:
On branch main

On branch feature/login



2)არის თუ არა ცვლილებები commit ისთვის მზად:

მაგ-“Changes to be committed”
ეს ნიშნავს: შენ უკვე გააკეთე git add ... და ეს ცვლილებები მზადაა commit-ისთვის.


3)ცვლილებები,რომლებიც ჯერ არაა დამატებული(unstaged)

მაგ-"Changes not staged for commit"
    ეს ნიშნავს,რომ ფაილები შეიცვალა მაგრამ git add ჯერ არ გაგიკეთებია რომ git ს აღექვა ცვლილება


4)ახალი ფაილები,რომლებსაც Git საერტოდ არ აკონტროლებს (untracked)

მაგ-"Untracked Files"-ეს ნიშნავს რომ, ეს ფაილები ახლახან არის შექმნილი და Git ჯერ ვერ აღიქვამს როგორც პროექტის ნაწილი



---------------------------------------------------------------------

- git add -> არის ბრძანება, რომელიც ირჩევს/ამზადებს კონკრეტულ ცვლილებებს შემდეგი commit ისთვის


Git ში მუშაობისას ცვლილებების შენახვა (commit) პირდაპირ არხდება. მანამდე არსებობს შუალედური ეტაპი,რომელსაც Staging Area(მოსამზადებელი ზონა ) ჰქვია. git add ს სწორედ ამ ზონაში გადააქვს შეცვლილი ან ჩამატებული ფაილები



მოდი ამ პროცესის მარტივი მაგალითი მოვიყვანოთ:

ვთქვათ რომ ვგზავნით რაღაც ამანათს:

1. Working Directory (სამუშაო საქაღალდე): ეს ოთახია, სადაც ნივთები(ფაილები), ყრია. ზოგს ვცვლით ზოგს ახალს ვქმნით.

2.Staging Area (git add) : ეს არის ყუთი. ჩვენ ვირჩევთ რომელი ნივთი ჩავდოთ ყუთში გასაგზავნად.ყველაფრის ჩადება არაა სავალდებულო

3.Repository (git commit): ეს არის ყუთის დალუქვა და ფოსტაში ჩაბარება



---------------------------------------------------------------------



- git diff -> არის თითქოს "იპოვე განსხვავება"(spot the difference) თამაში. მისი მთავარი დანიშნულებაა ზუსტად გვიჩვენოს, რა შეცვალე ფაილებში სიმბოლოების და ხაზების დონეზე

წინა მაგალითიდან, თუ git add არის ფაილების ყუთში ჩალაგება, git diff არის იმის შემოწმება თუ , რა დევს ამ ყუთში ან რა დაგვრჩა გარეთ.


-რას გვიჩვენებს git diff ის შედეგი?
    როდესაც ამ ბრძანებას გაუშვებ, ტერმინალში დაინახავ ტექსტს, რომელიც ასე იკითხება:

    -წითელი ფერი და  - ნიშანი ნიშნავს წაშლილ ხაზების
    -მწვანე ფერი და + ნიშანი ნისნავს დამატებულ ახალ ხაზებს.
    -თეთრი ტექსტი: კონტექსტი(უცვლელი ხაზები ცვლილების გარშემო),რომ მიხვდე სად მოხდა ცვლილება


 

-ძირითადი გამოყენების შედეგები:

    -git diff --> ადარებს სამუშაო საქაღალდეს Staging Areaს. გვიჩვენებს იმას, რაც შევცვალეთ, მაგრამ ჯერ არ დაგვიმატებია (git add)

    -git diff --staged --> ადარებს Staging Areaს და ბოლო commitს. გვიჩვენებს იმას რაც უკვე გაამზადე შესანახად

    -git diff HEAD --> ადარებს სამუშაო საქაღალდეს და ბოლო commit ს. გვიჩვენებს აბსოლუტურად ყველა ცვლილებას ბოლო შენახვის წერტილიდან.


    -git diff branch1 branch2 --> გვიჩვენებს სხვაობას ორ სხვადასხვა branch ს შორის.
