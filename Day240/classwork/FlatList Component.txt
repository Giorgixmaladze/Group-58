<FlatList> არის React Native-ის ერთ-ერთი ყველაზე მნიშვნელოვანი და მძლავრი ძირითადი კომპონენტი. ის შექმნილია მონაცემთა დიდი სიების (lists) მარტივად და, რაც მთავარია, ეფექტურად საჩვენებლად.


რაში გვჭირდება <FlatList>? (პრობლემა <ScrollView>-სთან)
წარმოიდგინეთ, რომ გაქვთ 1,000 კონტაქტისგან შემდგარი სია და მის გამოსაჩენად იყენებთ ჩვეულებრივ <ScrollView> კომპონენტს.


<ScrollView> მუშაობს ე.წ. "დაუყოვნებლივი" (Eager) რენდერინგის პრინციპით. ეს ნიშნავს, რომ ის 1,000-ივე კონტაქტის UI (მომხმარებლის ინტერფეისის) ელემენტს ერთდროულად ტვირთავს ტელეფონის მეხსიერებაში, მიუხედავად იმისა, ეკრანზე ამ მომენტში მხოლოდ 5 კონტაქტი ჩანს თუ 10. შედეგად, ტელეფონის ოპერატიული მეხსიერება (RAM) ივსება, აპლიკაცია იწყებს გაჭედვას და ელემენტის დასქროლვა ხდება შეუძლებელი.



აქ შემოდის <FlatList>:
ის იყენებს "ზარმაც" (Lazy) რენდერინგს. <FlatList> ხატავს მხოლოდ იმ ელემენტებს, რომლებიც ამჟამად ჩანს ეკრანზე (და რამდენიმეს რეზერვში მის ზემოთ და ქვემოთ). როდესაც სიას ქვემოთ სქროლავთ, ზემოთ დარჩენილი (უხილავი) ელემენტები იშლება მეხსიერებიდან და მათ ადგილს ახალი, ქვემოდან გამოჩენილი ელემენტები იკავებს. ამას მეხსიერების რეციკლირება ეწოდება, რაც აპლიკაციას უნარჩუნებს სისწრაფეს 100,000 ელემენტის შემთხვევაშიც კი.






როდის ვიყენებთ <FlatList>-ს?
ნებისმიერ დროს, როცა გვაქვს ერთგვაროვანი მონაცემების გრძელი ან ცვალებადი სია, რომელიც შეიძლება გაიზარდოს. მაგალითად:



-სოციალური ქსელების "Feed" (მაგ. Facebook ან Instagram პოსტები).


-კონტაქტების სია.


-პროდუქტების კატალოგი ელექტრონულ კომერციაში.


-შეტყობინებების (ჩატის) ისტორია.





მთავარი თვისებები (Props):

იმისათვის, რომ <FlatList>-მა იმუშაოს, მას აუცილებლად სჭირდება 3 ძირითადი თვისება (Prop):


1.data: თავად მონაცემები (მასივი - Array), რომლის ჩვენებაც გსურთ.


2.keyExtractor: ფუნქცია, რომელიც თითოეულ ელემენტს ანიჭებს უნიკალურ ID-ს. ეს React-ს ეხმარება სწრაფად იპოვოს და განაახლოს კონკრეტული ელემენტი სიის შეცვლისას.


3.renderItem: ფუნქცია, რომელიც იღებს data-ში არსებულ ერთ ელემენტს და აბრუნებს იმ UI კომპონენტს (მაგ: <Text>, <View>, ან თქვენს შექმნილ ბარათს), რომელიც ეკრანზე უნდა დაიხატოს.


მაგალითად:

import React from 'react';
import { SafeAreaView, View, FlatList, StyleSheet, Text } from 'react-native';

// 1. ჩვენი მონაცემები (სია)
const USERS_DATA = [
  { id: '1', name: 'გიორგი მაისურაძე', role: 'დეველოპერი' },
  { id: '2', name: 'ნინო ბერიძე', role: 'დიზაინერი' },
  { id: '3', name: 'დავით კახიძე', role: 'პროექტ მენეჯერი' },
  // ... წარმოიდგინეთ კიდევ 1000 მომხმარებელი
];

// 2. კომპონენტი, რომელიც ერთ კონკრეტულ მომხმარებელს დახატავს
const UserCard = ({ name, role }) => (
  <View style={styles.card}>
    <Text style={styles.name}>{name}</Text>
    <Text style={styles.role}>{role}</Text>
  </View>
);

const App = () => {
  return (
    <SafeAreaView style={styles.container}>
      <FlatList
        data={USERS_DATA} // ვაწვდით მონაცემებს
        keyExtractor={(item) => item.id} // ვეუბნებით, რომ უნიკალური გასაღები არის 'id'
        renderItem={({ item }) => ( // ვხატავთ თითოეულ ელემენტს
          <UserCard name={item.name} role={item.role} />
        )}
      />
    </SafeAreaView>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f5f5f5',
  },
  card: {
    backgroundColor: 'white',
    padding: 20,
    marginVertical: 8,
    marginHorizontal: 16,
    borderRadius: 8,
    shadowColor: '#000',
    shadowOpacity: 0.1,
    shadowRadius: 4,
    elevation: 2,
  },
  name: {
    fontSize: 18,
    fontWeight: 'bold',
  },
  role: {
    fontSize: 14,
    color: 'gray',
    marginTop: 4,
  },
});

export default App;
